01 · Overview

Cos'è Orientico

Una piattaforma SaaS che permette a qualunque attività locale (bar, ristorante, parrucchiere, pizzeria, negozio) di avere un sito web professionale online in 24 ore senza scrivere codice né collaborare con designer.

24h
Tempo di lancio
4
Stili visivi
minimal · classico · bold · elegante
49€
Setup una tantum
+ 9-19€/mese

🎯 Proposta di valore

L'attività locale compila un form di 6 campi, riceve via email 4 proposte di sito personalizzate generate da AI in pochi minuti, ne sceglie una, può chiedere modifiche in linguaggio naturale ("voglio più enfasi sull'aperitivo, aggiungi che siamo cane-friendly") e in 24 ore il sito è pubblicato. Aggiornamenti successivi via bot Telegram.

🔄 Rebranding da WebLocal → Orientico

Il progetto era inizialmente noto come MiniSito Local / WebLocal. È stato completamente rebrandizzato in Orientico (gioco di parole "orienta" + suffisso italiano), con nuovo logo (map pin arancione SVG), palette arancio caldo (#f97316) e crema (#fdf6ec), tipografia Instrument Serif italic per accenti emotivi.

★ Decisione architetturale
L'AI NON riscrive HTML. Riceve dati + richiesta cliente, restituisce SOLO valori testuali in JSON (hero_tagline, descrizione_breve, ecc.). Il template HTML resta intatto, intoccato. Vantaggi: prompt 10× più piccolo, zero rischio rottura layout, modifica istantanea con cache invalidation immediata.

04 · Landing

Landing page orientico.com

Design dark premium ispirato a Squarespace, con 10 effetti animati stratificati. Single-page, ~30KB gzipped, mobile-first.

🎬 10 animazioni stratificate

  • Cursore custom con lerp smooth + mix-blend-mode: difference (visibile su qualunque sfondo)
  • Mesh gradient animato: 3 blob arancioni in filter: blur(140px) con drift indipendenti (22s, 28s, 32s)
  • Hero text reveal word-by-word con overflow:hidden + translateY
  • Marquee gallery: 2 righe di template card che scorrono in direzioni opposte (50s e 60s)
  • Count-up numeri al primo scroll-in-view (200+, 24h, 49€, 0)
  • Magnetic buttons: il bottone segue il cursore con coefficiente 0.25
  • 3D tilt sulle pricing card (perspective 1200px, rotazione 6° max)
  • Scroll reveal via IntersectionObserver con threshold 0.15
  • Nav glassmorphism: backdrop-filter blur attivato a scrollY > 30px
  • Bento speed-meter: barra arancione che si riempie con cubic-bezier
★ Performance
Tutte le animazioni usano transform e opacity (accelerate GPU). IntersectionObserver attiva solo gli effetti visibili. Su mobile (@media(hover:none)) il cursore custom è disabilitato e i blob hanno opacità ridotta. Lighthouse Performance: 96/100.

★ Prompt strategy
Il prompt fornisce a Qwen i dati del cliente, i contenuti attuali e la richiesta di modifica. Qwen restituisce SOLO i campi modificati in JSON. Whitelist server-side blocca campi non ammessi (anti-injection). Risultato deserializzato e applicato via UPDATE Supabase. Email "anteprima aggiornata" inviata in fire-and-forget.
prompt-qwen-modifica.txt

- Nome: {{nome}}
- Categoria: {{categoria}}
- Stile scelto: {{stile}} # minimal | classico | bold | elegante

# CONTENUTI ATTUALI
{ "hero_tagline": ..., "descrizione_breve": ..., ... }

# RICHIESTA
"Voglio enfatizzare l'aperitivo serale, tono caldo ma sobrio"

# REGOLE
1. Mantieni tono coerente con stile {{stile}}
2. hero_tagline max 6 parole, d'effetto
3. NON inventare dati (orari, prezzi)
4. mostra_menu/galleria solo se richiesto esplicitamente

# OUTPUT
JSON valido, SOLO campi modificati, niente markdown.
08 · Scadenza

Sistema di scadenza

Due fasi di urgency per spingere conversione: 72h per scegliere uno stile, poi +24h per completare il pagamento.

📅 Fase 1: scelta (72h)

Dall'invio email, il cliente ha 72 ore per aprire le 4 anteprime e cliccare "Scegli questo". Banner countdown nel template (verde > 6h, arancione < 6h, rosso pulsante < 1h).

Allo scadere → redirect a /scaduta?fase=scelta

💳 Fase 2: pagamento (24h)

Dopo "Scegli", parte un nuovo timer di 24 ore per completare il pagamento. Premiamo l'engagement con più tempo decisionale, ma con maggiore urgency.

Allo scadere → redirect a /scaduta?fase=pagamento con CTA "Pubblica ora"

★ Decisione: computata, non memorizzata
La scadenza NON è una colonna del DB — è calcolata ad ogni request da created_at + 72h o scelta_stile_at + 24h. Vantaggio: cambiare policy (es. da 24h a 48h) è una sola riga di codice, nessuna migration o batch update. Le anteprime esistenti si adattano automaticamente. Niente cron job.

🚪 Pagina /scaduta — 3 azioni

Quando il cliente arriva all'URL di un'anteprima scaduta, viene reindirizzato a una pagina di rescue con tre opzioni:

  • Pubblica il sito (CTA primario) → checkout Stripe
  • Voglio modifiche → rigeneriamo le 4 anteprime gratis (form pre-compilato)
  • Parlami con qualcuno → modal con email/WhatsApp/telefono

Il testo del titolo si adatta alla fase: "Il tempo è scaduto" per fase 1, "Pronto a pubblicarlo?" per fase 2.

12 · n8n

Workflow n8n

Self-hosted n8n in Docker (n8n-weblocal su porta 5679). Un solo workflow attivo per Orientico.

📋 Workflow orientico-main-workflow

9 nodi, durata media 5-10s per richiesta:

  1. Webhook — riceve POST da /api/richiesta
  2. Normalizza Dati Cliente — Code node, pulisce telefono/email
  3. Qwen — Genera 4 Brief Creativi — HTTP Request a http://ollama:11434/api/chat
  4. Parsa Proposte AI — Code node, estrae JSON dai 4 brief
  5. Aggrega 4 Proposte — Code node, struttura finale
  6. Invia Email Orientico — HTTP Request a /api/send-anteprime/{id}
  7. Aggiorna DB — Avviso — HTTP Request a /api/notizie/{id} (parallelo)
  8. Risposta Webhook — Code node, JSON di success
⚠ Insight n8n versioning
n8n usa immutable versioning: ogni workflow ha un activeVersionId che punta a una entry in workflow_history. Modificare workflow_entity.nodes direttamente lascia la versione attiva orfana. Inoltre le connections referenziano nodi per nome, non per id — rinominare un nodo lo scollega silenziosamente. Lezione: modifiche raw DB richiedono di toccare TUTTI i livelli (nodes, connections, workflow_history).

14 · Sicurezza

Sicurezza & auth

🔑 Secrets

  • SUPABASE_SERVICE_KEY (wrangler.toml, env vars)
  • RESEND_API_KEY
  • ADMIN_SECRET = orientico-admin-2026
  • QWEN_API_KEY = "ollama" (placeholder, no auth)

🔒 Endpoint protetti

Endpoint AI/email richiedono header X-Admin-Secret:

  • /api/qwen-modifica/{id}
  • /api/send-anteprime/{id}
  • /api/send-modifica-applicata/{id}

Senza secret → 401. Test via curl confermato.

🛡 Anti-XSS

Tutti i template usano createElement + textContent (no innerHTML). Helper el() rende impossibile per costruzione XSS, anche se i contenuti dal DB fossero compromessi.

🚪 Whitelist Qwen

Qwen può modificare SOLO 8 campi della whitelist (CAMPI_AMMESSI). Anche se generasse {"is_admin": true}, viene scartato dal sanitizer prima dell'UPDATE.

⚠ Debito tecnico noto
Attualmente SUPABASE_SERVICE_KEY e RESEND_API_KEY sono in wrangler.toml committed (anche se la service key Supabase è in realtà l'anon key, non lo schema admin). Migrazione a wrangler pages secret put è raccomandata prima di pubblicare.

Prossimi passi

In ordine di priorità: cosa serve per chiudere il giro commerciale e iniziare ad acquisire clienti reali.

PRIORITÀ 1 Mappare designs JSON → campi AI

Il workflow n8n genera 4 brief creativi in designs (jsonb) ma NON li mappa in hero_tagline, descrizione_breve, ecc. Risultato: le anteprime aprono con i fallback ai campi base, perdendo la ricchezza del lavoro AI iniziale.

Fix proposto: aggiungere un step nel workflow n8n (dopo Qwen) che fa UPDATE Supabase con i campi più rappresentativi del primo brief, oppure rendere il template intelligente nel pescare da designs[0] se i campi nuovi sono null.

PRIORITÀ 2 Integrazione Stripe Checkout

Quando il cliente clicca "Pubblica il sito" (in fase pagamento o scaduta), serve un flusso di pagamento:

  • Endpoint /api/checkout/{id}: crea Stripe Checkout Session (49€ setup + abbonamento ricorrente)
  • Webhook /api/stripe-webhook: riceve checkout.session.completed → setta stato = pagata
  • Dopo pagamento → email "Sito in produzione entro 24h" + notifica admin

PRIORITÀ 3 Pubblicazione su dominio cliente

Quando stato = pagata, serve materializzare il sito su un dominio reale. Tre opzioni:

  • (a) Subdominio Orientico: {slug}.orientico.com (gratis, immediato)
  • (b) Dominio custom cliente: www.barcentrale.it (richiede ai clienti di puntare DNS)
  • (c) Acquisto domain via API Cloudflare Registrar (~10€/anno, valore aggiunto)

Soluzione tecnica: Pages Function di routing {slug}.orientico.com → renderizza il template {stile_scelto} con i dati del cliente.

PRIORITÀ 4 Aggiornamento sito via Telegram

Esiste già il workflow WebLocal — Bot Telegram v2 in n8n (legacy). Da riadattare al nuovo schema. Comandi target:

  • /avviso <testo> → setta promozione.testo
  • /orari <giorno> <range> → update orari JSONB
  • /menu → caricamento PDF/lista
  • /chiudi-oggi → setta avviso = "Chiusi per ferie"

PRIORITÀ 5 Form orari + categoria fine-grained

Attualmente il form chiede 6 campi base ma NON gli orari di apertura — vengono mostrati con placeholder default. Espandere il form:

  • Sezione "Quando siete aperti?" con 7 righe (Lun-Dom) e checkbox "stesso ogni giorno"
  • Categoria più granulare: invece di bar, scegliere bar-caffetteria / cocktail-bar / wine-bar per emoji + label più precise
  • Upload foto opzionale (1-3) → Supabase Storage

PRIORITÀ 6 Hardening sicurezza

  • Migrare SUPABASE_SERVICE_KEY, RESEND_API_KEY, ADMIN_SECRET da wrangler.toml a Cloudflare Pages Secret (wrangler pages secret put)
  • Rate limiting su /api/richiesta (max 5/ora per IP) — Cloudflare WAF rules
  • RLS Supabase: limitare letture su richieste al solo service role
  • Captcha turnstile opzionale sul form se spam dovesse aumentare

PRIORITÀ 7 Quality of life

  • Cloudflared come Windows service invece di container Docker manuale (resilienza a riavvii)
  • Modello LLM upgrade: passare da qwen2.5:7b a qwen2.5:14b o 32b — migliore qualità copywriting, latenza ancora accettabile
  • Dashboard admin a /dashboard: visualizza tutte le richieste, stato, scadenze, revenue
  • A/B test oggetto email + tasso conversione clic per stile
  • Analytics: aggiungere PostHog/Plausible su orientico.com per misurare conversione form
  • Esportazione sito statico: opzione "scarica come ZIP" per clienti che vogliono hosting proprio
💡 Decisione strategica aperta
Il modello attuale prevede setup 49€ + abbonamento 9-19€/mese. Vale la pena considerare anche un'opzione one-shot 199€ senza abbonamento (sito statico esportato, hosting cliente) per il segmento più tradizionale che diffida degli abbonamenti. Da validare con 5-10 clienti pilota.
